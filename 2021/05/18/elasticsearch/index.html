一、ElasticSearch介绍

1.1 引言

  1. 在海量数据数据中执行搜索功能时，传统数据库效率太低
  2. 如果关键字输入不准确，期望一样可以搜索到想要的数据
  3. 搜索的关键字高亮显示

1.2 ES的介绍

  1. ES是一个使用Java语言并且基于Lucene编写的搜索引擎框架，它提供了分布式全文搜索功能，提供一个统一的基于RESTful风格的WEB接口
  2. Lucene：本身就是一个搜索引擎的底层
  3. 分布式：ES主要是为了突出它的横向扩展能力
  4. 全文检索：对比模糊查询，将一段词语进行分词，并且将分出的单个词语统一放到一个分词库中，在搜索时，根据关键词去分词库中检索，找到匹配的内容(倒排索引)
  5. RESTful：操作ES很简单，只需要发送一个http请求，根据请求方式不同、携带参数不同，执行相应的功能
  6. 应用广泛：github、wiki、gold man

1.3 ES的由来

许多年前，一个刚结婚的名叫 Shay Banon 的失业开发者，跟着他的妻子去了伦敦，他的妻子在那里学习厨师。 在寻找一个赚钱的工作的时候，为了给他的妻子做一个食谱搜索引擎，他开始使用 Lucene 的一个早期版本。

直接使用 Lucene 是很难的，因此 Shay 开始做一个抽象层，Java 开发者使用它可以很简单的给他们的程序添加搜索功能。 他发布了他的第一个开源项目 Compass。

后来 Shay 获得了一份工作，主要是高性能，分布式环境下的内存数据网格。这个对于高性能，实时，分布式搜索引擎的需求尤为突出， 他决定重写 Compass，把它变为一个独立的服务并取名 Elasticsearch。

第一个公开版本在2010年2月发布，从此以后，Elasticsearch 已经成为了 Github 上最活跃的项目之一，他拥有超过300名 contributors(目前736名 contributors )。 一家公司已经开始围绕 Elasticsearch 提供商业服务，并开发新的特性，但是，Elasticsearch 将永远开源并对所有人可用。

据说，Shay 的妻子还在等着她的食谱搜索引擎…

1.4 ES和Slor

  1. Solr在查询死数据时，速度相对ES更快。但数据如果是实时改变的，Solr的查询速度会降低很多，ES的查询速度基本没有变化
  2. Solr需要基于Zookeeper来帮助集群搭建，ES自带集群概念
  3. 最开始Solr的社区非常火爆，针对国内文档不是很多。ES出现之后，ES的社区异常火爆且ES文档非常健全
  4. ES对现在的云计算和大数据支持的特别好

1.5 倒排索引

  1. 将存放的数据，以一定的方式进行分词，并将分词内容存放到单独的分词库中
  2. 当用户去查询数据时，会对用户的查询关键词进行分词
  3. 根据关键词的分词去分词库中匹配，最终得到数据的id标识
  4. 根据id标识去存放数据的位置拉取指定的数据

流程

  1. 客户端发送“谁中国”查询请求
  2. 对查询关键字进行分词得：谁、中国
  3. 查询分词库，匹配结果的数据id标识：1，2，3
  4. 返回对应id标识的数据

三、ElasticSearch基本操作

3.1 ES的结构

3.1.1 索引 Index

  1. ES的服务中心，可以创建多个索引
  2. 每个索引默认被分成5片存储
  3. 每一个分片都会存在至少一个备份
  4. 备份分片一般不参与检索数据，当ES检索压力大时会参与检索
  5. 备份的分片会放置在不同的节点中（单节点没有备份）
  6. 类似MySQL的数据库

3.1.2 类型 Type

  1. 一个索引下，可以创建多个类型
    1. ES 5.x 可以创建多个类型
    2. ES 6.x 可以创建一个类型
    3. ES 7.x 类型被废弃
  2. 相当于MySQL的表

3.1.3 文档 Document

一个类型下，可以有多个文档；这个文档相当于MySQL表中的列，存储实际数据

3.1.4 属性 Field

一个文档中可以包含多个属性，类型MySQL表中一行数据可以存在多个字段

3.2 RESTful语法

3.2.1 GET 请求

http://ip:port/index ：查询索引信息

http://ip:port/index/type/doc_id ：查询指定的文档信息

3.2.2 POST 请求

http://ip:port/index/type/doc_id/_search ：查询文档，可以在请求体中添加复杂的查询条件

http://ip:port/index/type/doc_id/_update ：修改文档，可以在请求体中添加修改的具体信息

3.2.3 PUT 请求

http://ip:port/index ：创建一个索引，需要在请求体中指定索引的信息、类型、结构

http://ip:port/index/type/_mappings ：创建索引时，指定索引文档存储的属性信息

3.2.3 DELETE 请求

http://ip:port/index ：删除索引

http://ip:port/index/type/doc_id ：删除指定的文档ß

3.4 Field的数据类型

3.4.1 字符型

  1. text：一般用于全文检索，将当前field进行分词
  2. keyword：不参与分词

3.4.2 数值型

  1. long：64位整数
  2. integer：32位整数
  3. short：16位整数
  4. byte：8位整数
  5. double：64位浮点数
  6. float：34位浮点数
  7. half_float：精度比float小一半，16位浮点数
  8. scaled_float：根据一个long和scaled来表示一个浮点型，如：long-123,scaled-100 = 1.23

3.4.3 时间型

date：支持三种时间格式：yyyy-mm-dd HH:mm:ss || yyyy-MM-dd || epoch_millis

3.4.4 布尔型

boolean：表示true/flase

3.4.5 二进制

支持Base64编码的字符串

3.4.6 范围型

  1. long_range：赋值时，无需指定具体具体内容，只需要存储一个范围即可，如gt、lt、gte、lte
  2. integer_range：同上
  3. double_range：同上
  4. float_range：同上
  5. date_range：同上
  6. ip_range：同上

3.4.7 经纬度

geo_point：用于存储经纬度，如定位

3.4.8 IP型

ip：支持IPV4和IPV6

3.4.9 其他

https://www.elastic.co/guide/en/elasticsearch/reference/7.10/mapping-types.html

3.5 创建带有结构的索引

RESTful语法

ps:es 7.x移除type，自动给定_doc作为type

# 创建带有结构的索引
PUT /book
{
  "settings": {
    "number_of_replicas": 1,
    "number_of_shards": 1
  },
  "mappings": {
    "properties": {
      "name":{
        "type": "text",
        "analyzer": "ik_max_word",
        "store": false,
        "index": true
      },
      "author":{
        "type": "keyword"
      },
      "count":{
        "type": "integer"
      },
      "on-sale":{
        "type": "date",
        "format": "yyyy-MM-dd HH:mm:ss||yyyy-MM-dd||epoch_millis"
      },
      "desc":{
        "type": "text",
        "analyzer": "ik_max_word"
      }
    }
  }
}

解释：

  1. number_of_replicas：分片备份数
  2. number_of_shards：分片数
  3. mappings：指定数据的结构
  4. properties：指定fields
  5. type：指定field类型
  6. analyzer：指定分词器
  7. store：是否额外存储，默认false
  8. index：是否作为查询条件，默认true

public class ESClient {

    private static RestHighLevelClient client;

    public static RestHighLevelClient getClient() {
        // 创建 HttpHost
        HttpHost httpHost = new HttpHost("localhost", 9200);

        // 创建 RestClientBuilder
        RestClientBuilder builder = RestClient.builder(httpHost);

        // 返回es客户端
        client = new RestHighLevelClient(builder);
        return client;
    }
    
    // 关闭客户端
    public static void close() throws IOException {
        if (client != null) {
            client.close();
        }
    }

}

4.2 创建索引

public void createIndex(String index) throws IOException {
    // 准备settings
    Settings.Builder settings = Settings.builder()
            .put("number_of_shards", 1)
            .put("number_of_replicas", 1);

    // 准备mappings
    XContentBuilder mappings = JsonXContent.contentBuilder()
            .startObject()
                .startObject("properties")
                    .startObject("name")
                        .field("type", "text")
                        .field("analyzer", "ik_max_word")
                    .endObject()
                    .startObject("age")
                        .field("type", "integer")
                    .endObject()
                .endObject()
            .endObject();

    // 创建索引
    CreateIndexRequest request = new CreateIndexRequest(index)
            .settings(settings)
            .mapping(mappings);

    CreateIndexResponse response = ESClient.getClient().indices().create(request, RequestOptions.DEFAULT);

    System.out.println(response.toString());

    ESClient.close();
}
